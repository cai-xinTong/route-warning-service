# 启动步骤

## 1. 创建数据库并初始化表结构

```bash
# 进入MySQL
mysql -u root -p

# 执行SQL脚本
source D:/ideaProjectNew/xianLuYUjing/route-warning-service/src/main/resources/sql/schema.sql
```

或者直接在MySQL客户端中执行 `schema.sql` 文件内容。

## 2. 修改数据库配置

编辑 `application.yml`，修改数据库用户名和密码：
```yaml
spring:
  datasource:
    username: root
    password: 你的密码
```

## 3. 编译项目

```bash
cd D:\ideaProjectNew\xianLuYUjing\route-warning-service
mvn clean package
```

## 4. 启动服务

### 方式1：使用Maven启动
```bash
mvn spring-boot:run
```

### 方式2：使用jar包启动
```bash
java -jar target/route-warning-service-1.0.0.jar
```

### 方式3：在IDE中启动
直接运行 `RouteWarningApplication.java` 的main方法

## 5. 导入Excel数据

启动成功后，调用接口导入路段经纬度数据：

```bash
curl -X POST "http://localhost:8080/api/import/excel?filePath=D:/ideaProjectNew/xianLuYUjing/路段经纬度.xlsx"
```

或使用Postman/浏览器访问：
```
POST http://localhost:8080/api/import/excel?filePath=D:/ideaProjectNew/xianLuYUjing/路段经纬度.xlsx
```

## 6. 测试预警功能

手动触发一次预警更新：
```bash
curl -X POST http://localhost:8080/api/warning/update
```

## 7. 查看预警结果

连接数据库查看预警信息：
```sql
SELECT * FROM warning_info ORDER BY create_time DESC;
```

## 8. 验证定时任务

等待6分钟，观察日志输出，确认定时任务自动执行。

## 常见问题

### Q1: 数据库连接失败
- 检查MySQL是否启动
- 检查用户名密码是否正确
- 检查数据库 `route_warning` 是否已创建

### Q2: Excel导入失败
- 检查文件路径是否正确（使用绝对路径）
- 检查Excel文件是否存在
- 检查文件格式是否为 `.xlsx`

### Q3: 网格数据接口调用失败
- 检查网络是否可访问 `http://172.22.96.169`
- 检查API Key是否正确
- 查看日志中的详细错误信息

### Q4: 没有生成预警信息
- 检查是否已导入路段节点数据
- 检查当前气象数据是否达到预警阈值
- 查看日志确认网格数据是否获取成功

## 日志位置

日志输出到控制台，可以通过以下方式查看：
- IDE控制台
- 终端窗口
- 如需输出到文件，可在 `application.yml` 中配置

## 端口占用

默认端口：8080

如需修改，编辑 `application.yml`：
```yaml
server:
  port: 8081
```
