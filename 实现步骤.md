# 路段预警改造实现步骤

## 背景
原来从 `geo_line_node` 表直接获取所有路段节点。
现在改为：先查 `weather_service_plan`（status=1）→ 获取 elements 和 GeoIds → 按 GeoIds 查 `weather_service_geo_detail` → 对每个站点按 elements 指定的要素计算预警。

---

## 实现步骤

- [x] **步骤1：新建 `WeatherServicePlan` 实体类**
  - 对应表 `weather_service_plan`，字段：id, departmentId, elements, GeoIds, status, type, planName

- [x] **步骤2：新建 `WeatherServiceGeoDetail` 实体类**
  - 对应表 `weather_service_geo_detail`，字段：staId, lon, lat, geoId, stationName, stationCode

- [x] **步骤3：新建 `WeatherServicePlanMapper`**
  - 继承 BaseMapper，提供查询 status=1 的 plan 列表

- [x] **步骤4：新建 `WeatherServiceGeoDetailMapper`**
  - 继承 BaseMapper，并新增 `selectByGeoIds(List<Long> geoIds)` 方法（XML 中用 IN 查询）

- [x] **步骤5：新建 `WeatherServiceGeoDetailMapper.xml`**
  - 实现 selectByGeoIds 的 SQL（`WHERE geoId IN <foreach>`）

- [x] **步骤6：修改 `WarningScheduleService.updateWarnings()`**
  - 原逻辑：`geoLineNodeMapper.selectList(null)` 取全量节点，对所有节点做全要素预警
  - 新逻辑：
    1. 查 `weather_service_plan` 中 status=1 的记录（可能多条）
    2. 对每条 plan，解析 GeoIds（按逗号分割转 List<Long>）
    3. 按 GeoIds 查 `weather_service_geo_detail` 得到站点列表
    4. 解析 elements（按逗号分割，如 `WIND,VISIBILITY,RAIN`）
    5. 仅对 elements 包含的要素调用对应 WarningService.checkWarning()
    6. 将产生的预警入库（保持原有批量插入逻辑）

- [x] **步骤7：新增 `GeoDetailNode` 适配层（或复用）**
  - `checkWarning()` 接收的是 `GeoLineNode`，而新数据源是 `WeatherServiceGeoDetail`
  - 在 `WarningScheduleService` 中新增 `toGeoLineNode()` 私有方法，将 `WeatherServiceGeoDetail` 的 lon/lat/stationCode/stationName 映射到 `GeoLineNode`

- [ ] **步骤8：联调测试**
  - 确认 plan 查询、GeoDetail 查询、要素过滤、预警入库全流程正常
