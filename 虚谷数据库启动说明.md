## 虚谷数据库特殊说明

### 1. 主键生成
虚谷数据库不支持AUTO_INCREMENT，使用序列生成主键：
- `seq_geo_line_node.NEXTVAL`
- `seq_warning_threshold.NEXTVAL`
- `seq_warning_info.NEXTVAL`

### 2. 数据类型映射
- `DATETIME` → `TIMESTAMP`
- `JSON` → `VARCHAR(2000)`
- `AUTO_INCREMENT` → 序列

### 3. 索引创建
索引需要单独创建，不能在CREATE TABLE中内联。

### 4. 注释
使用 `COMMENT ON TABLE/COLUMN` 语法添加注释。

## 常见问题

### Q1: 连接数据库失败
- 检查网络是否可访问 `10.159.90.202:36034`
- 检查用户名密码是否正确
- 检查虚谷数据库服务是否启动

### Q2: 序列不存在
确保执行了 `schema_xugu.sql` 脚本，其中包含序列创建语句。

### Q3: 插入数据失败
检查序列是否正常工作：
```sql
SELECT seq_geo_line_node.NEXTVAL FROM DUAL;
```

### Q4: 时间字段格式问题
虚谷数据库的TIMESTAMP格式可能与MySQL不同，如有问题可调整Java代码中的时间处理。

## 修改阈值

阈值存储在数据库中，可随时修改：
```sql
-- 修改暴雨黄色预警阈值为35mm
UPDATE warning_threshold
SET threshold_value = 35
WHERE warning_type = 'RAIN' AND level_name = 'YELLOW';

COMMIT;
```

修改后需要重启服务，或等待服务重启时自动加载新阈值。
